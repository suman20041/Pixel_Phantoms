PROJECT: PHANTOM_NODE

>> DIFFICULTY: BEGINNER | EST. COST: $4.50 | TIME: 45 MINS

This protocol will guide you through constructing a basic IoT sensor node capable of reporting temperature data to the cloud via MQTT. This is the fundamental unit of the Pixel Phantoms hardware network.

// 01_ACQUIRE_HARDWARE

  • ESP8266 (NodeMCU or Wemos D1 Mini) ~$3.00
  • DHT11 Temperature Sensor ~$1.00
  • Jumper Wires (Female-to-Female) ~$0.50
  • Micro-USB Cable (Existing)

// 02_WIRING_DIAGRAM

Connect the components as follows. Ensure the board is disconnected from power while wiring. 

ESP8266      DHT11
------------------
3V3 (or 5V)  -> VCC (+)
GND          -> GND (-)
D1 (GPIO 5)  -> DATA (Signal)
WARNING: Double-check your VCC and GND connections. Reversing polarity will fry your sensor instantly.

// 03_FLASH_FIRMWARE

1. Install the Arduino IDE.
2. Add ESP8266 board manager URL in Preferences.
3. Install the DHT sensor library by Adafruit. 


#include <ESP8266WiFi.h>
#include <DHT.h>

#define DHTPIN D1
#define DHTTYPE DHT11

DHT dht(DHTPIN, DHTTYPE);
const char* ssid = "YOUR_WIFI_NAME";
const char* password = "YOUR_WIFI_PASS";

void setup() {
  Serial.begin(115200);
  dht.begin();
  WiFi.begin(ssid, password);
}

void loop() {
  float t = dht.readTemperature();
  Serial.print("Temp: ");
  Serial.println(t);
  delay(2000);
}

// 04_DEPLOYMENT

Upload the code and open the Serial Monitor (115200 baud). You should see temperature readings streaming in. You have successfully initialized a Phantom Node.
